# 周界网关 — 上位机北向 TCP HEX 协议

本文档供**上位机软件**开发使用，描述上位机与开发板上网关程序（`perimeter-gateway`）之间的通信协议。

网关负责将上位机指令转发至继电器设备，并将设备原始 HEX 回包封装后返回上位机。

---

## 1. 连接方式

| 项目 | 说明 |
|------|------|
| 协议 | TCP |
| 连接模式 | **长连接**（建立一次连接，可连续发送多条指令） |
| 目标地址 | 开发板 **eth1** 网口 IP |
| 默认端口 | `9000`（可在网关 `config.yaml` 中修改） |
| 字节序 | 多字节字段均为 **大端（Big Endian）** |
| 示例地址 | `192.168.1.201:9000` |

> 上位机需与开发板 eth1 在同一网段（如 `192.168.1.0/24`）。

---

## 2. 帧格式

### 2.1 公共约定

| 字段 | 长度 | 值 | 说明 |
|------|------|-----|------|
| STX | 2 字节 | `AA 55` | 帧头，固定 |
| VER | 1 字节 | `01` | 协议版本 |
| CRC16 | 2 字节 | — | Modbus CRC16，见 §2.4 |

### 2.2 请求帧（上位机 → 网关）

```
┌──────┬─────┬─────┬──────────┬──────────┬─────────┐
│ STX  │ VER │ CMD │ LEN      │ PAYLOAD  │ CRC16   │
│ AA55 │ 01  │ 1B  │ 2B (BE)  │ N 字节   │ 2B (BE) │
└──────┴─────┴─────┴──────────┴──────────┴─────────┘
```

- **LEN**：PAYLOAD 长度，当前所有指令 PAYLOAD 为空（`LEN = 0x0000`）
- **CRC 覆盖范围**：从 VER 到 PAYLOAD 末尾（不含 STX 和 CRC 本身）

### 2.3 响应帧（网关 → 上位机）

```
┌──────┬─────┬─────┬────────┬──────────┬──────────┬─────────┐
│ STX  │ VER │ CMD │ STATUS │ LEN      │ PAYLOAD  │ CRC16   │
│ AA55 │ 01  │ 1B  │ 1B     │ 2B (BE)  │ N 字节   │ 2B (BE) │
└──────┴─────┴─────┴────────┴──────────┴──────────┴─────────┘
```

- **CMD**：与请求相同，用于配对
- **STATUS**：处理结果，见 §3
- **PAYLOAD**：南向设备原始 HEX 回包（成功时）；失败时通常为空

### 2.4 CRC16 算法

采用 **Modbus CRC16**：

```
初始值 crc = 0xFFFF
对每个字节 b（从 VER 到 PAYLOAD 末字节）：
    crc ^= b
    重复 8 次：
        若 crc 最低位为 1：crc = (crc >> 1) ^ 0xA001
        否则：           crc = crc >> 1
最终 crc 以 Big Endian 写入帧尾
```

## 3. 状态码 STATUS

| 值 | 名称 | 说明 |
|----|------|------|
| `0x00` | 成功 | PAYLOAD 为设备原始回包 |
| `0x01` | 参数错误 | 请求 PAYLOAD 格式不合法（如设置 IP 时长度不对） |
| `0x02` | 设备超时 | 继电器未在超时时间内响应 |
| `0x03` | 设备不可达 | TCP 连接失败或连接断开 |
| `0x04` | 回包格式错误 | 设备有响应，但 HEX 不符合协议文档 |
| `0x05` | 网络配置失败 | 读写 eth1.network 或 networkctl 应用失败 |
| `0xFF` | 未知命令 | CMD 不在支持列表中 |

## 4. 命令列表

### 4.1 周界拦截（设备 A — FE 协议）

| CMD | 功能 | 请求帧（完整 HEX） |
|-----|------|-------------------|
| `0x01` | 开启电源 | `AA 55 01 01 00 00 18 50` |
| `0x02` | 关闭电源 | `AA 55 01 02 00 00 18 A0` |
| `0x03` | 信号打开 | `AA 55 01 03 00 00 D8 F1` |
| `0x04` | 信号关闭 | `AA 55 01 04 00 00 19 40` |
| `0x05` | 查询开关闭状态 | `AA 55 01 05 00 00 D9 11` |

**成功时 PAYLOAD（设备原始回包）：**

| CMD | 期望 PAYLOAD |
|-----|-------------|
| `0x01` | `FE 05 00 07 FF 00 29 F4` |
| `0x02` | `FE 0F 00 00 00 08 40 02` |
| `0x03` | `FE 0F 00 00 00 07 00 06` |
| `0x04` | `FE 0F 00 00 00 07 00 06` |
| `0x05` | 见下表 |

**查询开关闭状态（0x05）PAYLOAD 解析：**

| PAYLOAD（示例） | 含义 |
|----------------|------|
| `FE 01 01 FF xx xx` | 信号**已打开** |
| `FE 01 01 00 xx xx` | 信号**未打开** |
| `FE 01 01 80 xx xx` | 信号**未打开**（与 `00` 相同） |

> 判断规则：帧头 `FE 01 01` 固定，第 4 字节 `FF`=开，`00` 或 `80`=关。

### 4.2 附加功能（设备 B — Modbus 01 协议）

| CMD | 功能 | 请求帧（完整 HEX） |
|-----|------|-------------------|
| `0x11` | 温度查询 | `AA 55 01 11 00 00 DD 51` |
| `0x12` | 空调开启 | `AA 55 01 12 00 00 DD A1` |
| `0x13` | 空调关闭 | `AA 55 01 13 00 00 1D F0` |

**成功时 PAYLOAD（设备原始回包）：**

| CMD | 期望 PAYLOAD |
|-----|-------------|
| `0x11` | `01 03 02 HH LL CRC CRC`（见温度换算） |
| `0x12` | `01 06 00 15 00 01 59 CE` |
| `0x13` | `01 06 00 15 00 00 98 0E` |

**温度换算（0x11）：**

```
原始值 = (HH << 8) | LL        // 大端 uint16
温度(℃) = 原始值 - 40

示例：01 03 02 00 47 F8 76
      0x0047 = 71 → 71 - 40 = 31℃
```

### 4.3 开发板网络（eth1 IP 管理）

| CMD | 功能 | 请求 PAYLOAD | 响应 PAYLOAD |
|-----|------|--------------|--------------|
| `0x20` | 设置 eth1 IP | 5 字节，见下表 | 成功时回显设置的 5 字节 |
| `0x21` | 查询 eth1 IP | 无（`LEN=0`） | 5 字节，见下表 |

**IP PAYLOAD 格式（5 字节）：**

| 偏移 | 长度 | 说明 |
|------|------|------|
| 0 | 4 字节 | IPv4 地址（大端按字节顺序，如 `192.168.1.201` → `C0 A8 01 C9`） |
| 4 | 1 字节 | 掩码长度（如 `/24` → `0x18`） |

**设置 IP 请求示例（改为 `192.168.1.100/24`）：**

```
AA 55 01 20 00 05 C0 A8 01 64 18 <CRC16>
                 └─ LEN=5
                       └─ IP + prefix
```

**查询 IP 请求示例：**

```
AA 55 01 21 00 00 <CRC16>
```

**查询 IP 成功响应示例（`192.168.1.201/24`）：**

```
AA 55 01 21 00 00 05 C0 A8 01 C9 18 <CRC16>
                  ^^ STATUS=00
                        ^^^^^ LEN=5
```

**查询 IP 数据来源（0x21）：**

1. 优先读取 `eth1.network` 配置文件中的 `Address=x.x.x.x/yy`（IP 与 prefix 同源）
2. 文件不存在或解析失败时，回退到网卡当前 IPv4 地址与掩码
3. 再回退时默认 prefix 为 `/24`

> **注意：** 设置 IP 成功后，systemd-networkd 会应用新地址，但网关进程仍绑定在**旧 IP** 上监听 `9000` 端口。需执行 `systemctl restart perimeter-gateway`，上位机再用**新 IP** 重连。若 `networkctl` 应用失败，网关会**回滚** `eth1.network` 并返回 `STATUS=0x05`。

## 5. 响应示例

### 5.1 成功 — 开启电源（0x01）

```
请求: AA 55 01 01 00 00 18 50
响应: AA 55 01 01 00 00 08 FE 05 00 07 FF 00 29 F4 62 DB
                  ^^ STATUS=00
                        ^^^^^ LEN=8
                              └─ PAYLOAD（设备回包）
```

### 5.2 成功 — 查询状态，信号已打开（0x05）

```
请求: AA 55 01 05 00 00 D9 11
响应: AA 55 01 05 00 00 06 FE 01 01 FF 21 DC 13 74
```

### 5.3 成功 — 查询状态，信号未打开（0x05）

```
响应（00）: AA 55 01 05 00 00 06 FE 01 01 00 61 9C 13 74
响应（80）: AA 55 01 05 00 00 06 FE 01 01 80 60 3C 13 74
```

### 5.4 成功 — 温度查询 31℃（0x11）

```
请求: AA 55 01 11 00 00 DD 51
响应: AA 55 01 11 00 00 07 01 03 02 00 47 F8 76 1B 93
                              └─ PAYLOAD: 01 03 02 00 47 F8 76
```

### 5.5 成功 — 空调开启（0x12）

```
请求: AA 55 01 12 00 00 DD A1
响应: AA 55 01 12 00 00 08 01 06 00 15 00 01 59 CE 99 1E
```

### 5.6 失败 — 设备超时（0x02）

```
响应: AA 55 01 01 02 00 00 FC B9
                  ^^ STATUS=02, LEN=0, 无 PAYLOAD
```

### 5.7 失败 — 设备不可达（0x03）

```
响应: AA 55 01 01 03 00 00 3C E8
```

### 5.8 失败 — 回包格式错误（0x04）

```
响应: AA 55 01 05 04 00 04 FE 01 01 10 2B 13
                  ^^ STATUS=04
                        ^^^^^ LEN=4
                              └─ 收到的异常原始数据（便于调试）
```

## 6. 通信流程

```
上位机                          网关                         继电器
  │                              │                              │
  │──── TCP Connect ────────────►│                              │
  │                              │                              │
  │──── 请求帧 (CMD) ───────────►│──── 南向 HEX ───────────────►│
  │                              │◄─── 设备原始回包 ────────────│
  │◄─── 响应帧 (STATUS+PAYLOAD) ─│                              │
  │                              │                              │
  │──── 下一条请求帧 ───────────►│  ...（同一 TCP 连接）         │
  │                              │                              │
  │──── TCP Close ──────────────►│                              │
```

## 7. 上位机实现要点

### 7.1 粘包处理

TCP 是字节流，一次 `read` 可能收到半帧或多帧。建议：

1. 在缓冲区中查找 `AA 55` 帧头
2. 读取 LEN 字段，计算整帧长度
3. 校验 CRC16，通过后解析
4. CRC 错误或帧头不对时，丢弃并继续对齐下一帧

**请求帧长度** = `6 + LEN + 2`（无 PAYLOAD 时为 8 字节）

**响应帧长度** = `7 + LEN + 2`

### 7.2 超时建议

| 阶段 | 建议超时 |
|------|----------|
| TCP 连接 | 3 ~ 5 秒 |
| 等待单条响应 | 10 ~ 15 秒（网关内部含设备重试，默认最多约 9 秒） |

### 7.3 请求-响应配对

- 通过 **CMD 字段**配对请求与响应
- 建议同步模式：发一条等一条，避免并发混乱
- 若需异步，请维护 CMD + 序号映射

### 7.4 CRC 伪代码

```c
uint16_t crc16_modbus(const uint8_t *data, size_t len) {
    uint16_t crc = 0xFFFF;
    for (size_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (int j = 0; j < 8; j++) {
            if (crc & 1)
                crc = (crc >> 1) ^ 0xA001;
            else
                crc >>= 1;
        }
    }
    return crc;
}

// 请求帧 CRC 输入：data = [VER, CMD, LEN_H, LEN_L, ...PAYLOAD]
// 响应帧 CRC 输入：data = [VER, CMD, STATUS, LEN_H, LEN_L, ...PAYLOAD]
```

## 8. 快速联调

使用 netcat 发送单条指令（短连接测试）：

```bash
# 查询开关闭状态
printf '\xAA\x55\x01\x05\x00\x00\xD9\x11' | nc -q2 192.168.1.201 9000 | xxd

# 温度查询
printf '\xAA\x55\x01\x11\x00\x00\xDD\x51' | nc -q2 192.168.1.201 9000 | xxd

# 查询开发板 eth1 IP
printf '\xAA\x55\x01\x21\x00\x00\xD2\x51' | nc -q2 192.168.1.201 9000 | xxd

# 设置开发板 eth1 IP 为 192.168.1.100/24（成功后需重启网关）
printf '\xAA\x55\x01\x20\x00\x05\xC0\xA8\x01\x64\x18\x56\x5E' | nc -q2 192.168.1.201 9000 | xxd
```

## 9. 附录：命令与南向协议对照

详细南向设备协议见 [周界拦截.md](./周界拦截.md)。上位机只需使用本文档中的北向 CMD，无需直接拼接南向 FE / Modbus 帧。

| 北向 CMD | 网关转发目标 | 南向设备 |
|----------|-------------|----------|
| `0x01` ~ `0x05` | 周界拦截继电器 | 设备 A（config: `perimeter`） |
| `0x11` ~ `0x13` | 空调/温度继电器 | 设备 B（config: `environment`） |
| `0x20` ~ `0x21` | eth1 IP 读写 | 本地 `/etc/systemd/network/eth1.network` |
